# AI, Copilot, and MCP notes

## Use order
1. Lock in human rules for source / ref / test / docs / CI first.
2. Add AI acceleration second.
3. Keep contracts, tests, and owners as the final guardrails.

## Quick guidance
- **Copilot**: good for first drafts of docs, tests, metrics, and SQL.
- **Local MCP**: best for local coding agents that need governed project context.
- **Remote MCP**: useful for application builders who want managed access to Semantic Layer / discovery style tools.

## Caution
Never treat generated code as trusted just because it compiles. Review it against your project conventions, data contracts, and tests.
